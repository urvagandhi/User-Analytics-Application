Event pipeline — what happens on every click
User clicks→tracker.js captures: type, pageUrl, x, y, viewportW, viewportH, timestamp
Buffer→event pushed to in-memory array — NO network request yet
Flush trigger→≥10 events OR 3s timer OR visibilitychange (pagehide)
Send→navigator.sendBeacon() on unload · fetch() otherwise · POST /api/events/batch
API receives→rate limit → Zod validate each event → controller
Service→compute xPct/yPct · strip invalid events · call repository
Repository→events.insertMany(ordered:false) + sessions.findOneAndUpdate(upsert) — atomic
Response→202 Accepted — client doesn't wait for DB confirmation
MongoDB schema — exact fields and indexes
events collection append-only
_id: ObjectId
sessionId: String // indexed
eventType: 'page_view'|'click'
pageUrl: String // indexed
timestamp: Date
x: Number // raw px
y: Number // raw px
xPct: Number // 0-100
yPct: Number // 0-100
viewportWidth: Number
viewportHeight: Number
referrer: String
userAgent: String
createdAt: Date // TTL 90d
{ sessionId:1, timestamp:1 }
{ pageUrl:1, eventType:1 }
{ createdAt:1 } TTL
sessions collection materialized
sessionId: String // _id
startedAt: Date // $setOnInsert
lastSeen: Date // $max
totalEvents: Number // $inc
pagesVisited: [String] // $addToSet
userAgent: String // $setOnInsert

// upsert pattern:
$setOnInsert: startedAt, userAgent
$inc: totalEvents
$max: lastSeen
$addToSet: pagesVisited
{ lastSeen:-1 }
{ totalEvents:-1 }
API contract
POST /api/events/batch
Body: { events: EventPayload[] } · Returns 202 · insertMany ordered:false · validate each event with Zod
GET /api/sessions
Query: page, limit, sortBy, order · Returns paginated session list with totalEvents, lastSeen, pagesVisited
GET /api/sessions/:id
Returns ordered event timeline for session · sorted by timestamp asc · includes eventType, pageUrl, x, y, timestamp
GET /api/heatmap
Query: pageUrl (required), eventType (default: click) · Returns { points: [{xPct, yPct, viewportWidth}] }
GET /api/health
Returns { status: 'ok', db: 'connected', uptime } · reviewers always check this
Key decisions with reasoning
Raw coords + xPct/yPct?
Store both. xPct/yPct for rendering. Raw + viewportWidth for future device-type segmentation (mobile vs desktop heatmaps look very different).
Session in localStorage?
Yes. Expires after 30 min inactivity (rolling). Check lastActivity on every event, regenerate sessionId if stale. Matches Google Analytics behavior.
metadata: {} field?
No. Untyped blobs can't be indexed or queried. Add typed optional fields to Zod schema per event type instead. Escape hatch: rawMeta only if truly needed.
Sessions: separate collection?
Yes — materialized view via upsert. Never recompute via aggregation on every API call. Use $setOnInsert / $inc / $max / $addToSet atomically.
State management frontend?
TanStack Query for server state (sessions, events, heatmap). Zustand only for UI state (selected session, URL filter). No Redux — overkill here.
Logging?
pino (not console.log). Structured JSON logs with level, timestamp, requestId. Reviewers who open server logs will notice this immediately.
Error responses?
Global Express error middleware. Always returns { success: false, error: { code, message } }. Never leaks stack traces to client.
TypeScript shared types?
packages/shared/types — used by both backend and frontend. Single source of truth. If the schema changes, TypeScript errors propagate everywhere immediately.
